# Mutation bench — 2026-07-07 baseline

First run of `defn-bench --mutations-only`. Harness lives at
`cmd/defn-bench/mutations.go`; each case exercises one Phase C projection
op end-to-end (agent invoked via `claude -p`, verified against a
whitespace-canonicalized post-condition on the final file).

- defn commit at time of run: `3a6c0c2` (v0.25.0 + Phase C followups shipped)
- defn binary: freshly `go install`'d from `main` before the run.
- claude: whichever was on `$PATH` at run time (system default `claude` CLI).
- Fixture files are single-file, ~5-10 line Go functions written fresh
  into a scratch git repo per case (`git reset --hard` between modes).

## Per-case results

| Case                    | files calls | defn calls | files time | defn time | files ✓ | defn ✓ |
|-------------------------|------------:|-----------:|-----------:|----------:|:-------:|:------:|
| insert-precondition     |          15 |          8 |      2m02s |     1m06s |   ✓     |   ✓    |
| wrap-in-defer           |          11 |         10 |      1m42s |     1m28s |   ✓     |   ✓    |
| rename-param            |          11 |          9 |      1m13s |     1m07s |   ✓     |   ✓    |
| replace-slice-return    |          11 |          9 |      1m23s |     1m04s |   ✓     |   ✓    |
| add-import              |           6 |         12 |        56s |     1m11s |   ✓     |   ✓    |
| **TOTAL**               |    **54**   |   **48**   |   **7m15s**|  **5m56s**|  5/5    |  5/5   |

**Aggregate: 11% tool-call reduction; 18% wall-time reduction; 100%
correctness in both modes.**

## Reading the shape

- **insert-precondition** — the strongest defn win (15 → 8 calls, 46%
  wall-time reduction). One `code(op:"insert-precondition", ...)` replaces
  Read + Edit + fmt-check + re-Read confirmation.
- **wrap-in-defer / rename-param / replace-slice-return** — modest wins
  (~20% call reduction each). The fixtures are so short that Read+Edit is
  nearly a fixed 2-3 call chain; defn saves the semantic-refactor tax but
  not much else.
- **add-import LOSES on this fixture** (6 → 12 calls) — but the reason
  is environmental, not op-related. On follow-up runs I captured the
  full `stream-json` trace and saw:
  - **The defn-mode agent NEVER called `code(op:"add-import", ...)`**.
    It tried `ToolSearch(select:mcp__defn__code)`, got no match, tried
    three more queries, gave up, and fell back to `Edit`.
  - **Root cause:** `claude -p` headless mode did not load the scratch
    dir's `.mcp.json` — even after adding `--mcp-config .mcp.json` and
    committing the file to survive `git clean -fdq` (v1 harness didn't;
    the file was silently wiped after case 1). The MCP defn tool was
    never actually available in any case's defn-mode run.
  - **Files-mode inflation:** the global `~/.claude/hooks/enforce-defn-
    on-go-edits.sh` hook fires because `defn init` creates `.defn/` — so
    files-mode Edit gets blocked once, the agent Reads the hook script
    to understand why, Bashes to inspect, then retries Edit. That's ~3
    calls of pure harness overhead added to files-mode's total.
  - **Every v1 number is a noise-vs-noise comparison.** Both modes had
    the MCP tool absent AND both modes paid a hook tax. Defn-mode was
    ALSO paying repeated-ToolSearch tax on top. The 11% "reduction"
    aggregate isn't measuring what it claims.

## What this run does NOT measure

- **Real-repo size** — every fixture is a fresh scratch file, so
  files-mode never pays the Read cost of a 500-line real file. The token
  and call delta on real code will be larger (roughly proportional to
  file size for read-heavy modes).
- **Ranking / retrieval interplay** — no read-side questions were run.
  Real workflows blend read + edit; this bench isolates edit only.
- **Multi-mutation coherence** — each case is a single-op edit. Chains
  of ops (rename a param, then wrap a stmt in defer) may amplify defn's
  edge because state stays in the DB rather than being re-Read on each
  step.
- **Fixture bias** — the prompts were written to sound neutral, but the
  defn-mode preamble names the exact op that solves each case. That's
  a hint even if not a directive. A truer test would run both modes
  with only a generic tools-you-have preamble.

## Delta artifacts to look at next

1. Redo add-import with a **larger** fixture (10-15 imports, mixed
   stdlib + third-party). Files-mode should struggle more; defn-mode's
   goimports-canonical grouping should shine.
2. Add a **multi-op coherence** case (rename then wrap) to see if the
   files-mode Read tax stacks.
3. Vary the defn-mode preamble to a **generic** "use defn for Go" line —
   remove the per-op explanations — to see how much of defn's win is
   naming vs semantics.
4. Capture **token usage** (currently only tool calls + wall time).
   `stream-json` includes `usage` on assistant messages.

## Verdict (revised after trace analysis)

**v1's numbers do not measure what they claim.** The MCP tool was
absent in every defn-mode run; the harness distortion from a wiped
`.mcp.json` and a firing global hook is unbounded. Do not cite the
"11% reduction" or the 4/5-wins framing anywhere — treat this as a
noise floor, not a baseline.

The harness has been hardened (see commit hardening `runMutationBench`
+ `runMutationCase`):

1. `.mcp.json` and `CLAUDE.md` are now committed after `defn init` so
   `git clean -fdq` between cases can't wipe them.
2. `claude -p` is invoked with `--mcp-config .mcp.json` to force the
   scratch dir's MCP server to load in headless mode.
3. `CLAUDE_ALLOW_GO_EDIT=1` is set in the child env so the global
   `enforce-defn-on-go-edits.sh` hook doesn't fire inside the bench.
4. `git add` calls use explicit paths (never `-A` / `.`) so the harness
   can't accidentally include something the scratch dir picked up.
5. `mutationDefnPreamble` now instructs the agent to `ToolSearch(select:
   mcp__defn__code)` as its first step, so the deferred tool is loaded
   before the task starts.

A re-run against the hardened harness is the actual v1 baseline. This
file records the pre-hardening state as a witness to how the
measurement got there.
